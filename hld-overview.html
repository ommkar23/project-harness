High-Level Design

Project Harness: Evaluation and Prompt Improvement System

This project is a lightweight evaluation harness for prompt experiments. Its job is to test prompt behavior against a small benchmark, score the results, and use the evaluation feedback to refine the prompt until quality improves or a fixed loop limit is reached.

System Intent

Primary Goal

Create a controlled environment for measuring prompt quality instead of relying on intuition or one-off manual testing.

Current Use Case

Evaluate startup-headline generation for brevity, required language, and banned language across multiple prompt candidates.

Operating Model

Run a benchmark, score outputs, identify failures, generate a better prompt, and repeat within a capped optimization loop.

Execution Flow

1

Seed Evaluation

The harness begins with the current prompt candidates and runs the benchmark set against Gemini.

2

Score Aggregation

Each output is evaluated using fixed rules. The system calculates pass counts and average scores to establish a ranked view of prompt quality.

3

Failure Compression

Failed cases are summarized into a compact explanation of what went wrong, such as excessive verbosity or missing required language.

4

Prompt Improvement

Gemini receives the evaluation feedback and proposes a revised system prompt aimed at improving the next evaluation round.

5

Loop Control

The new prompt is re-evaluated. The loop stops when quality no longer improves or when the maximum of 5 iterations is reached.

Current Boundaries

What It Covers Today

  • Single-task prompt evaluation
  • Single-provider Gemini execution
  • Deterministic scoring
  • Prompt self-improvement loop
  • Stored run artifacts for later inspection

What It Does Not Cover Yet

  • Human review workflows
  • Tool-using agents or agent trajectory evaluation
  • Multi-provider routing and comparison
  • Dashboarding or persistent experiment database
  • Production orchestration or scheduling

Design Summary

In HLD terms, this is a closed-loop prompt evaluation system. The core value is not raw model access, but the ability to benchmark outputs, identify failure patterns, improve prompts systematically, and stop with a measurable best-so-far result.

This page intentionally avoids implementation details and code structure. It is meant to explain the system as a product and architecture concept.
